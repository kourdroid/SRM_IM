## 2024-06-13 - Inline render functions with FlatList cause excessive re-renders
**Learning:** React Native's `FlatList` component is highly optimized but re-renders its entire list if the `renderItem` function is re-created on every render. This happens frequently in functional components where `renderItem` is defined inline. In our offline-first app where many items might be listed, this significantly degrades UI performance when lists get large. Furthermore, helper functions used in `renderItem` should be moved out of the component if they do not depend on state or props to reduce memory allocation during render cycles.
**Action:** Always wrap `renderItem` functions for `FlatList` in a `useCallback` hook, and ensure all references are correctly included in the dependency array (which helps ESLint's `react-hooks/exhaustive-deps` rule). Additionally, move pure helper functions like formatting utilities out of the component scope entirely to eliminate unnecessary function recreations.
