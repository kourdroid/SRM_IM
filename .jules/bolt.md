## 2024-05-18 - FlatList optimizations with useCallback
**Learning:** React Native `FlatList` component will trigger unnecessary re-renders of its list items when the `renderItem` function is defined inline within the component scope. This happens because the function reference changes on every render of the parent component.
**Action:** Always wrap `renderItem` and helper functions that it uses (like `parseMediaUrls`, `formatIncidentDate`, etc.) in `useCallback` or move them outside the component if they do not depend on component state. Since `FlatList` is a pure component, providing a stable reference to `renderItem` will avoid re-rendering list items unless the data changes. For inline render functions, ESLint requires that all referenced parent scope variables (like state setters `setSelectedIncident`, `setIsModalVisible`) are included in the dependency array.
