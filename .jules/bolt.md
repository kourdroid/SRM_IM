## 2024-05-18 - FlatList inline functions\n**Learning:** Passing inline callback functions or non-memoized inner functions to React Native's `FlatList` (such as `onEndReached`, `renderItem`) creates new function references on every render cycle. This negates `FlatList`'s built-in optimizations because it interprets the new function reference as a prop change, triggering unnecessary re-renders of the list and its items.\n**Action:** Always wrap inner functions used for `FlatList` rendering or callbacks (like `renderItem`, `onEndReached`, `keyExtractor`) with `useCallback`. Be sure to include all necessary dependencies (like state variables or setters from the component scope) in the dependency array to avoid stale closures and satisfy the ESLint `react-hooks/exhaustive-deps` rule.
