## 2024-06-10 - Optimizing FlatList Render Stability
**Learning:** In React Native, pure helper functions declared inside a component (like `formatIncidentDate`) and inline render functions (like `renderItem` for `FlatList`) cause new function allocations on every component render. This forces the `FlatList` to re-render all visible items unnecessarily, causing scrolling jank and performance degradation, especially in complex UIs.
**Action:** Always proactively hoist pure helper functions (e.g., date formatters, parsing utilities) outside the component scope to prevent unnecessary function re-allocations. Wrap `renderItem` functions in `useCallback`, ensuring all referenced parent scope variables (like state setters) are included in the dependency array to satisfy ESLint's `react-hooks/exhaustive-deps` rule and maintain reference stability.
