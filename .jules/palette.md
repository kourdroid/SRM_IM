## 2024-05-18 - Standardize Type and Status Badges
**Learning:** Type badges and status badges are styled inconsistently across different screens (`app/(tabs)/home.tsx`, `app/(admin)/dashboard.tsx`, `app/(admin)/incidents.tsx`, `app/(director)/incidents.tsx`). They often apply background colors and padding directly to `<Text>` components, or use hardcoded styles, which violates the `SrmStatusBadge` component usage and the Memory instruction: "semantic badges like typeBadge must use a distinct <View> container for box-model styles... rather than applying them directly to <Text> components."
**Action:** Unify all status and type badges by either replacing them with `SrmStatusBadge` from `@/components/ui/srm` or ensuring they use a `View` container with the appropriate `badgeVariants` styles when a custom layout is needed. For this PR, we'll replace the hardcoded type badges with a unified `SrmTypeBadge` pattern or by reusing `SrmStatusBadge` with a custom variant or just standardizing `typeBadge` styles to use `View` containers properly.
