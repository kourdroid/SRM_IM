## 2024-05-24 - PostgREST .or() String Injection Vulnerability
**Vulnerability:** Unsanitized user input (`filters.search`) was being directly interpolated into a Supabase `.or()` query string: `.or(\`description.ilike.%${filters.search}%,village.ilike.%${filters.search}%\`)`. Because PostgREST parses this string to build the query, characters like `,`, `.`, `(`, and `)` in the user input can alter the structure of the query, leading to query injection or syntax errors.
**Learning:** Supabase JS's `.or()` method, when taking a string argument, evaluates it as a raw PostgREST filter string. It is not parameterized like object-based filters or standard `.eq()` calls. Directly interpolating unfiltered user input is equivalent to SQL injection in a PostgREST context.
**Prevention:** Always sanitize user input before interpolating it into a Supabase `.or()` string by removing or escaping PostgREST syntax characters (e.g., `.replace(/[,.()]/g, '')`), or prefer using parameterized object-based filtering if the library version supports it.
